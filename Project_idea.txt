Statistical API for traffic measurement data of the city of Helsinki
Traffic measurements from Helsinki which contain data of traffic volume, average speeds and the amounts of different types of vehicles, collected for different means of transportation (vehicles, pedestrians, and cyclists). The REST API provides statistical data in both JSON and CSV format, for multiple data sources and different means of automatic collection, e.g. LAM stations in the Helsinki region and pedestrian counting devices.

The update frequency and aggregation interval vary by data source, of which more detailed descriptions can be fetched using the metadata operations of the API. Other information which can be queried includes the locations of the different counting devices and their aggregated, statistical measurement history.


Key endpoints used
Endpoint	Purpose
GET /api/metadata/sources/	List all data sources
GET /api/counters/	List all sensors with coordinates
GET /api/observations/	Raw observations per sensor
GET /api/observations/aggregate/	Aggregated data by period

Helsinki Traffic Analysis — Project Brief
Statistical analysis of traffic measurement data from the City of Helsinki using the Lidotiku REST API. The project fetches sensor observations, samples statistically representative data, and analyses traffic patterns by hour, day, and month.

Project overview
The City of Helsinki operates 3,569 traffic sensors across the metropolitan area, collecting vehicle counts, average speeds, and pedestrian/cyclist volumes. This project queries the public REST API, draws a statistically valid random sample, downloads aggregate observations, and extracts key traffic patterns.

Key findings:

Busiest hour: 13:00 (622 avg vehicles) — lunch hour dominates over morning rush
Busiest day: Wednesday (9,683 avg) — Friday dips significantly, likely hybrid work
Busiest month: May (352,650 avg) — anomaly under investigation; all other months cluster between 142k–241k

Address information of buildings in the built environment information system
RThe built environment information system, Ryhti, brings together information on construction and land use nationwide. The information is generated in the basic work of the municipalities and provincial associations, and it is delivered to the national information system primarily through the interface. The construction information to be submitted to the system is a construction permit, demolition permit, landscape work permit and deviation decision. The addresses of the objects are delivered with the information. Addresses are created in Ryhti&#39;s structured form. In the first phase, the addresses do not have their own location points, but are assigned to building sites.
Construction information is available nationwide from the built environment information system. The address information of buildings created before the introduction of the built environment information system has been entered into the built environment information system from the population information system of the Digi and population information system. During the transition period (2025-2028), municipalities can still submit the construction permit information to the population information system, from where it is forwarded to the built environment information system. The data content of the building data of the built environment information system is described in the interoperability platform: https://tietomallit.suomi.fi/model/raklu .
